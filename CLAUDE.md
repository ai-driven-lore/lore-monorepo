## Project Status

Lore is currently in the **specification phase**. The package format, API contracts, and architecture may change until the first stable release. No application code exists yet — only documentation and architecture specifications under `.docs/`.

## What is Lore?

Lore is a cloud-native platform for packaging, versioning, validating, and distributing AI knowledge (Skills, Rules, Specs, Templates) across development teams and organizations. Inspired by Docker Hub and Helm Registry, it provides a standardized, runtime-agnostic distribution layer for AI assets.

Key principle: **a Lore Package is runtime-agnostic**. It must never contain platform-specific files (e.g., `SKILL.md`, `AGENTS.md`, Cursor-specific configs). Platform-specific artifacts are generated by the CLI at install time based on the detected AI runtime.

## Core Domain Concepts

- **Skill** — reusable capability teaching an AI how to perform a specific task
- **Rule** — behavioral constraints and policies
- **Spec** — structured project requirements
- **Template** — reusable project scaffolding
- **Namespace** — publisher identity (e.g., `m1cloud`, `community`, `microsoft`)

## Package Format

**Identifier:** `<namespace>/<name>:<version>` — e.g., `m1cloud/azure-architect:1.0.0`

**Distribution unit:** `*.lore` compressed archive — e.g., `azure-architect-1.0.0.lore`

Every package must include a `manifest.yaml`:

```yaml
apiVersion: lore.io/v1
kind: Skill           # Skill | Rule | Spec | Template
metadata:
  namespace: m1cloud
  name: azure-architect
  version: 1.0.0
spec:
  title: Azure Architect
  description: Azure architecture specialist
content:
  defaultLanguage: en-US
translations:
  - en-US
  - pt-BR
dependencies:
  - m1cloud/terraform-reviewer
license: Apache-2.0
tags:
  - azure
```

## Validation Rules

The Validation Engine enforces: metadata correctness, schema compliance, dependency resolution, prompt injection detection, and required translation presence. Invalid packages cannot be published.
